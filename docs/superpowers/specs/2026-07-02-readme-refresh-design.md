# SmartPages README 改版设计

## 目标

将中英文 README 从偏长的功能手册改为产品推广优先的项目首页，让首次访问者迅速理解 SmartPages 的价值、看到真实演示，并能在几分钟内完成安装和首次使用。

## 受众与原则

- 第一受众是寻找网页操作录制和 AI 文档生成工具的新用户。
- 第二受众是准备了解实现、参与开发或评估安全性的开发者。
- 先展示价值和效果，再说明安装与配置，最后承接开发、贡献和许可信息。
- 只描述仓库中已经可用的能力，不扩张产品承诺。
- 中英文保持相同信息结构，但分别使用自然文案，不做逐句机械翻译。

## 信息架构

1. 首屏：语言切换、图标、核心价值主张、简短说明、技术与许可徽章。
2. 演示：保留 `docs/assets/smartpages-demo-zh.gif`，紧跟首屏文案展示完整工作流。
3. 核心价值：用三个短模块概括自动捕获、AI 成文、编辑与导出。
4. 工作方式：用简短步骤说明从录制到交付文档的流程。
5. 安装与快速开始：优先给出最短可执行路径，再补充开发构建方式。
6. 能力与兼容性：精炼说明文档类型、模型提供商、输出格式与自定义能力。
7. 产品界面：保留 `docs/assets/readme-hero.png`，集中说明主要界面。
8. 隐私与安全：说明 API Key 存储、内容发送边界、CSP 和 HTML 清理措施。
9. 开发与贡献：保留常用命令、专项文档入口、贡献说明和许可证。

## 内容取舍

- 删除重复出现的功能介绍、工作流描述和状态表。
- 将完整架构、目录树、配置字段解释等开发细节压缩为入口链接或简短摘要。
- 保留安装、快速开始、模型兼容、隐私安全、贡献和双许可证等影响采用决策的内容。
- 不宣传当前并未内置的 PDF 直接导出；如需提及，只说明可通过 HTML 打印为 PDF。

## 视觉与文案

- 演示 GIF 位于首屏价值主张之后，静态产品图位于后续“产品界面”部分。
- 使用短段落、三列价值点、有限表格和清晰标题，减少连续长列表。
- 中文核心文案围绕“录一次操作，自动写成清晰文档”；英文采用含义一致的自然表达。
- 保持现有仓库内图片和徽章，避免引入外部设计依赖。

## 验证

- 检查中英文 README 的章节、事实、链接和安装命令一致。
- 检查所有仓库内相对链接与图片路径存在。
- 对照 `manifest.json`、`package.json` 和实现代码核对版本、命令、支持格式及模型能力。
- 确保 Markdown 在 GitHub 中可正常渲染，中文文件保持 UTF-8 编码。
