# 示例：用户无法收到验证码处理流程

> 适用场景：客服或技术支持团队录制排查流程，使用 SmartPages 生成知识库条目。

## 问题描述

用户反馈登录或绑定手机号时无法收到短信验证码。

## 适用范围

该流程适用于以下情况：

- 用户手机号格式正确，但长时间未收到验证码。
- 用户多次点击发送验证码后仍未收到短信。
- 客服需要在后台检查验证码发送记录。

## 处理步骤

### 1. 确认用户手机号

在工单详情中查看用户填写的手机号，并确认国家区号和手机号格式是否正确。  
[截图1]

### 2. 进入短信发送记录

打开客服后台，进入“消息中心”中的“短信发送记录”。  
[截图2]

### 3. 搜索手机号

在手机号搜索框中输入用户手机号，点击“查询”。  
[截图3]

### 4. 查看发送状态

检查最近 10 分钟内是否存在验证码短信记录，并查看发送状态。  
[截图4]

### 5. 判断处理方式

如果状态为“发送成功”，建议用户检查短信拦截、手机信号和收件箱容量。  
如果状态为“发送失败”，记录失败原因并转交技术支持。  
如果没有发送记录，检查用户是否触发了发送频率限制。  
[截图5]

## 推荐回复话术

您好，我们已经帮您检查短信发送记录。请先确认手机信号、短信拦截和收件箱容量。如果仍无法收到验证码，我们会继续为您排查发送通道状态。

## 需要升级的情况

- 同一手机号持续发送失败。
- 多个用户在同一时间段集中反馈收不到验证码。
- 后台显示短信通道异常或供应商返回失败。
