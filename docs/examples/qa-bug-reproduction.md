# 示例：后台筛选条件重置失败复现说明

> 适用场景：QA 在测试后台系统时，使用 SmartPages 录制真实复现流程，再生成可提交给研发的复现文档。

## 问题概述

在订单管理页面中，用户设置筛选条件后点击“重置”，页面清空了输入框，但订单列表仍保留筛选后的结果。预期行为是筛选条件和列表结果同时恢复到默认状态。

## 环境信息

| 项目 | 内容 |
| --- | --- |
| 页面 | 订单管理 |
| 浏览器 | Chrome |
| 账号角色 | 运营管理员 |
| 录制来源 | SmartPages 浏览器插件 |

## 复现步骤

1. 打开后台系统并进入“订单管理”页面。  
   [截图1]

2. 在“订单状态”下拉框中选择“待发货”。  
   [截图2]

3. 在“下单时间”中选择最近 7 天，并点击“查询”。  
   [截图3]

4. 确认列表只展示待发货订单。  
   [截图4]

5. 点击筛选区域右侧的“重置”按钮。  
   [截图5]

6. 观察页面：筛选控件已恢复默认，但列表数据仍然是待发货订单。  
   [截图6]

## 实际结果

筛选条件显示已清空，但列表结果没有刷新，仍保留上一次查询结果。

## 预期结果

点击“重置”后，筛选条件和订单列表都应恢复到默认状态。

## 补充说明

该问题可能影响运营人员批量查单和导出数据。建议检查“重置”按钮是否只清理了表单状态，而没有触发列表重新请求。
