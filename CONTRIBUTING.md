# 贡献指南 (Contributing Guide)

感谢你对 SmartPages 的关注！欢迎提交 Issue、Pull Request 或建议。

## 重要：贡献者许可协议 (CLA)

**提交 Pull Request 即表示你同意以下条款：**

1. **版权转让**：你同意将所提交代码的所有权利（包括但不限于著作权、商业使用权）转让给项目所有者（汪鸿儒 / Hongru Wang）。

2. **授权许可**：如果你所在地区的法律不支持版权转让，你同意授予项目所有者对所提交代码的**独占性、永久性、不可撤销的、全球性的、免费的**许可，包括：
   - 使用、复制、修改、合并、发布、分发、再授权、销售
   - 将你的代码以任何协议（包括商业协议）再授权给第三方
   - 以你的代码为基础创作衍生作品

3. **原创保证**：你保证所提交的代码是你原创的，或你有权提交这些代码且不会侵犯任何第三方的知识产权。

4. **无额外义务**：项目所有者没有义务使用、保留或署名你所提交的代码。

## 如何贡献

### 提交 Issue
- 描述问题或建议功能
- 提供复现步骤（如果是 Bug）

### 提交 Pull Request
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/your-feature`)
3. 提交修改 (`git commit -m 'Add some feature'`)
4. 推送到分支 (`git push origin feature/your-feature`)
5. 创建 Pull Request

**提交 PR 前请确保：**
- 代码通过 ESLint 检查 (`npm run lint`)
- 代码通过类型检查 (`npm run typecheck`)
- 已阅读并同意上方的贡献者许可协议 (CLA)

### 代码规范
- 遵循项目现有代码风格
- 添加必要的注释
- 保持修改范围最小化
